Note: The original feed is served without an Access-Control-Allow-Origin response header, so it cannot be used directly from another origin per CORS restrictions. I have tried a few work-arounds and settled on two. Here is a summary of everything I tried:

  • Failed: automatically loading the original feed - Browsers do not allow access to remote feeds without an Access-Control-Allow-Origin response header unless one uses the no-cors option, which makes the feed inaccessible via JavaScript.
  • Failed: automatically loading an unmodified local cache - Browsers do not allow automatic JavaScript access to local files.
  • Rejected: automatically loading a local cache as a script with the contents assigned to a variable - That worked, but I prefer to show that I can work with unmodified feeds.
  • Working: prompting the user to manually select a local file then using that - This works well enough for development purposes, but I don't think it is the best solution for the challenge.
  • Work in Progress: automatically loading a cache of the orignal feed from a differnet server - I have saved a cache of the feed to the GitHub repository for my challenge solution. It should appear both in the repo and in the published site.
